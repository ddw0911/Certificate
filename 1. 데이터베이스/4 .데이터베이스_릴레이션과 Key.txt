# 관계 DB의 구조
? Relation
- 데이터를 표 형태로 표현한 것
- Relation Schema(구조를 나타내는), Relation Instance(실제 값)으로 구성

? Tuple
- 릴레이션을 구성하는 행
- Tuple의 수 : Cardinality

? Attribute
- DB를 구성하는 가장 작은 논리 단위
- 개체의 특성 기술
- Attribute의 수 : Degree

? Domain
- 하나의 Attribute가 취할 수 있는 같은 타입의 원자 집합

? Relation Schema
- 릴레이션의 논리적 구조를 기술
- 정적인

? Relation Instance
- 구체적인 데이터 값 (extension)
- 동적인

? Extension : 일정한 개념이 적용되는 사물의 범위

# Relation의 특징
- 튜플, 속성 간 순서는 없다
- 릴레이션을 구성하는 튜플 식별을 위해 속성을 부분집합지어 키(Key)로 설정

# Key의 개념 및 종류
1. 후보키
- 기본키로 사용 가능한 속성들 (속성들의 부분집합)
- 릴레이션의 모든 튜플에 대해 유일성, 최소성을 만족시켜야

2. 기본키 PK
- 특정 튜플을 구별할 수 있는 '유일한' 속성 - 레코드 검색or정렬에 이용
- Null 불가, 속성값 중복 불가
ex) 학번, 주민번호

3. 대체키
- 기본키를 제외한 나머지 후보키

4. 슈퍼키
- 속성들의 집합으로 구성된 키
ex) 학번+ 주민번호 등
- 모든 튜플에 대해 유일성 만족 but 최소성 x

5. 외래키 FK
- 참조되는 릴레이션의 기본키와 대응되어 참조관계를 표현
ex) 어떤 릴레이션 R1의 기본키의 값과 일치함을 요구하는 다른 릴레이션 R2의 한 속성

# 무결성
1. 개체 무결성 : 기본키를 구성하는 속성은 NULL 값, 중복값을 가질 수 없음
2. 참조 무결성 : 외래키 값은 NULL이거나 참조 릴레이션의 PK와 동일해야함
3. 도메인 무결성
