## 정보시스템 -조직체에 필요한 Data를 수집해두었다 필요시 정보를 생성
ex) 인사행정, 군사정보, 경영정보, 의사결정지원 시스템

- 자료, 정보
자료 - 수집한 사실 (미가공)
정보 - 유용한 형태 (가공) /w 자료처리시스템

? 자료처리시스템 
? 데이터웨어 하우스 - 조직의 업무 시스템에서 추출된 것으로 생성된 데이터베이스

## 데이터베이스란
통합된 데이터 - 자료 중복 배제
저장된 데이터 - 실시간 접근 가능
운영 데이터 - 조직 업무에 필요한 
공용 데이터 - 공동 소유

## 데이터베이스의 특성
실시간 접근성 - 실시간 응답 가능
계속 변화 - 삽입, 삭제, 갱신으로 최신데이터 유지
동시 공유 - 동시 이용 가능
내용에 의한 참조 - 내용으로 데이터 검색 (주소,위치 x)

## 기존 파일시스템의 문제점
종속성 - 데이터와 app이 상호의존적 -> 저장방법or접근방법을 변경 시 app도 같이 변경
중복성 - 일관성(데이터간 불일치), 보안성(동등한 보안 유지 어려움), 경제성(저장공간낭비 + 비용증가), 무결성(정확성 유지 어려움)
-> 해결하기 위한 데이터베이스 구축

## DBMS의 정의와 기능
DB관리 + 사용자 요구에 따른 정보 생성
기능 3
1. 정의(조직) - 데이터의 타입, 구조, 조건 명시
2. 조작 - 데이터 검색, 갱신, 삽입, 삭제를 위한 접근수단(인터페이스) 제공
3. 제어 - 데이터에 접근 시 무결성 유지 + 데이터 접근에 대한 보안유지, 권한검사 + 동시 접근 시 병행수행 제어

## DBMS의 장단점
단점
- 전산화 비용, 전문가 필요
- 대용량 저장소에 집중 Access로 인한 오버헤드
- 복잡한 시스템 + 백업, 복원 어려움

장점
- 중복제거 + 일관성, 무결성, 보안유지
- 데이터를 표준화 + 통합관리 + 최신화유지
- 공동 이용 + 실시간 처리
- 논리적, 물리적 독립성
