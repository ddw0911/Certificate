# 데이터모델 표시사항 - 구조 연산 제약조건

# 데이터모델 구성요소
1. 개체 - 현실의 대상체
2. 속성 
- 데이터의 가장 작은 논리적 단위
- 개체의 항목
? 속성들이 가질 수 있는 모든 값의 집합 - 도메인
3. 관계 - 개체or속성 간의 관계

# 개체-관계 모델
- 개념적 데이터 모델 중 대표적
- ER 다이어그램으로 유형제한 없는 표현가능

# ER 다이어그램
- 아이디어를 시각적으로 표현하기 위한 도구
- 사각형(개체) 마름모(관계) 타원(속성) 밑줄타원(기본키) 등

# 계층형 데이터 모델
- 논리적 구조도가 tree형태, 개체가 구성노드
- 개체 간 상속 관계로 표현 -> 레코드 삭제 시 연쇄삭제
- 1:N 관계만 존재 + 개체(세그먼트) 간 사이클 불가
ex) IMS

# 망(그래프, 네트워크) 데이터 모델
- 그래프를 이용해 논리구조 표현
- N:M 관계 (상위레코드 Owner, 하위레코드 Member) + 레코드 간 1:1, 1:N 가능
ex) DBTG, EDBS, TOTAL

# 데이터베이스 설계
1. 개념적 설계
- 현실세계를 추상적으로 표현 /w ER다이어그램
- 개념 스키마 모델링, 트랜잭션 모델링 병행
2. 논리적 설계
- 물리적 저장장치에 저장하기 위해 DBMS가 지원하는 구조로 변환
- 개념 스키마 평가 및 정제, 트랜잭션 인터페이스 설계
3. 물리적 설계
- 물리적 구조의 데이터로 변환
- 데이터베이스 파일의 저장구조 및 액세스경로 결정
- 고려사항 : 반응시간, 공간활용도(저장공간의 양), 트랜잭션 처리량

# 데이터베이스 설계 순서
1. 요구 분석
2. 개념적 설계 : 개념스키마, 트랜잭션 모델링, ER모델
3. 논리적 설계 : DBMS에 맞는 스키마 설계, 트랜잭션 인터페이스 설계
4. 물리적 설계 : DBMS에 맞는 물리적 구조의 데이터로 변환
5. 구현 : DDL로 데이터베이스 생성, 트랜잭션 작성
