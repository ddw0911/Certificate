## 스키마(schema) (메타데이터)
DB 구조, 제약조건을 기술 - 개체entity 속성attribute 관계realtionship 등
- 사용자 관점에 따라 : 외부, 개념, 내부 스키마
- 데이터 사전에 저장
? 데이터 사전 : DB에 저장된 모든 정보를 관리 (시스템 카탈로그)

# 스키마 3계층
1. 외부스키마 (사용자View)
- 개인 입장에서 필요한 DB의 논리적 구조 정의 (서브스키마)
- 하나의 DB에 여러개 가능
- 같은 DB에 대해 다른 관점 허용
- 사용자는 SQL로 DB사용

2. 개념스키마 (전체View)
- DB의 전체적인 논리구조
- 모든 프로그램이나 사용자(기관,조직)들이 필요한 DB (하나만 존재)
- 개체 간 관계, 제약조건 + 접근권한, 무결성규칙 정의
- DB 관리자가 구성

3. 내부스키마
- DB의 물리적 구조
- 데이터의 표현방법, 저장순서
- 프로그래머, 시스템 설계자 관점의 스키마

# 데이터베이스 언어
1. 데이터 정의언어 (DDL)
 - DB를 구축 or 수정 목적
 - 데이터 사전에 여러개 테이블로 저장
 - 기능
  1. 외부스키마 정의
  2. DB 정의 및 수정
  3. 제약 조건 정의
  4. 데이터의 물리적 순서 규정

2. 데이터 조작 언어 (DML) (서브언어)
 - 사용자를 위한 도구
 - 호스트 언어에 DB 기능 추가
 - 대표적으로 Query - 터미널에서 이용하는 비절차적 언어

3. 데이터 제어 언어 (DCL)
 - 데이터 보호, 관리 목적 - 무결성, 보안, 회복, 병행수행
